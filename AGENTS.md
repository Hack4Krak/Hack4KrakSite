## Overview

**Monorepo** using **Bun** as package manager (Node runtime):
- **Frontend**: Nuxt (TypeScript + Vue)
- **Backend**: Rust (Actix-web + SeaORM)

Root `package.json` defines workspace tasks that delegate to each package.

## Directory Structure

```
frontend/             # Nuxt frontend application
  ├── ARCHITECTURE.md # Frontend architecture details
  ├── DEVELOPMENT.md  # Development setup guides
  └── package.json    # Frontend package configuration
backend/              # Rust backend application
  ├── src/            # Main application code
  │   ├── routes/      # HTTP handlers (define OpenAPI schema)
  │   ├── services/    # High-level abstractions
  │   └── models/      # Database logic
  ├── migration/      # Database migrations
  ├── ARCHITECTURE.md # Backend architecture details
  ├── DEVELOPMENT.md  # Backend setup & commands
  └── Cargo.toml      # Rust package configuration
package.json          # Root package with workspace commands
```

## Agent Commands

Commands you'll commonly use:

- **Linting**: `bun lint:fix` - format and lint all packages
- **Testing**: `bun test` - run unit and integration tests; `bun frontend:e2e` - run frontend e2e tests (requires frontend running); `bun backend:test:full` - run backend full test suite
- **OpenAPI**: `bun backend:openapi-cli -- write` - regenerate API specification from code

## Key Architecture Patterns

- **OpenAPI Contract**: Backend routes define the API schema. Frontend types are auto-generated from this contract and stored in `frontend/openapi/`. Regenerate after backend schema changes using our cli. Never try to recreate OpenAPI schema manually or using self-written scripts .
- **Database Entities**: Auto-generated by SeaORM into `backend/src/entities/`. See `backend/DEVELOPMENT.md` for migration and entity generation workflows.
